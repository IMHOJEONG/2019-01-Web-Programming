부분적으로 웹 페이지를 갱신할 수 있는 기술 
=> 크롬 개발자 도구의 Network 탭에서 확인할 수 있음 
=> ex) 내용만이 담겨있는 파일을 받아올 수 있음 
=> 페이지를 전환할 때마다 모든 파일을 받는 것 대신 부분적으로 변경하는 것이 나음
=> 사용성의 증대(변경할 수 있는 부분에 집중 가능, 서버 뿐만 아닌 사용자 입장에서 돈, 자원 절약
가능)
=> Ajax를 배우는 이유

수작업으로 웹 페이지 수정은 한계에 봉착 => Ajax를 통해 기본적 HTML 구조를 재사용하고 
바뀔 수 있는 부분과 고정되는 부분을 구분해 바뀔 수 있는 부분만 Ajax로 동적으로 변동시켜
주면 됨.

ex) 1억개를 동시에 바꿀 수 있는 장점이 존재 => 각각 페이지마다 html 코드를 가질 필요 없고
compact 한 효과를 가능
